# v2.2.2

This is a stability and release-confidence patch.

It does not change the writing model. It makes the installer, packaging, and long-term maintenance story much more trustworthy.

## Highlights

- installer progress is now stateful and explicit
  - `State: copying bundled app files`
  - `State: checking PATH integration`
  - `State: verifying installed binary`
  - `State: opening post-install options`
  - `State: post-install summary is ready`
- installer PATH messages are clearer
  - if nothing had to be written, it now says so directly instead of leaving a vague pause after `PATH update already present in shell config`
- installer debug mode is now available
  - `./install.sh --debug`
  - one-line bootstrap form: `curl ... | bash -s -- --debug`
- release/install regression coverage is much stronger
  - packaged install smoke
  - public `curl | bash` smoke
  - existing-install update rerun
  - installer action matrix for `about`, `doctor`, `repair-path`, `uninstall`, and `factory-reset`

## Why this release exists

The app itself was stable.

The remaining risk was the product surface around it:

- installer states that felt like hangs
- update paths that only failed in specific combinations
- release confidence living partly in memory instead of in scripts and docs

`v2.2.2` closes that gap.

## Install

Latest stable:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash
```

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash -s -- --version v2.2.2
```

