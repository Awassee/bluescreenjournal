# BlueScreen Journal v0.1.13

This release is a focused UX polish pass for global search.

The goal was to make search faster to drive from the keyboard while keeping the nostalgic screen style and no-plaintext-on-disk guarantees unchanged.

## Install

Turnkey macOS install:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash
```

Pin this release:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash -s -- --version v0.1.13
```

## Improvements

- live search typing now keeps focus on the active input field instead of unexpectedly jumping into the results list
- manual search execution (`Enter`) still jumps to results immediately, preserving quick open behavior
- added new range presets in search:
  - `W` for last 7 days
  - `Y` for year-to-date
- added search overlay keyboard accelerators:
  - `Ctrl+N` and `Ctrl+P` to move result selection
  - `Ctrl+L` to clear query, filters, and results
  - `Ctrl+R` to recall the most recent query
- search range filters now persist between overlay opens
- selected result is retained across reruns when the same result is still present
- search menu now includes explicit week/year scope actions for discoverability
- search overlay guidance now reflects the new controls and selection state

## Why this matters

- global search feels less jumpy during query editing
- repeated search workflows require fewer keystrokes
- keyboard-only users can refine, navigate, and rerun searches without mode friction
