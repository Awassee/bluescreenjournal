# v2.2.0

This release is about making BlueScreen Journal easier to learn and easier to trust.

It keeps the nostalgic writing core intact while improving three things around it:

1. first-run guidance
2. release certification
3. visible nostalgia QA review

## Highlights

- added a new `First 2 Minutes Cheat Sheet` guide:
  - `bsj guide cheatsheet`
  - `HELP -> First 2 Minutes Cheat Sheet`
  - installer option `4`
- wired the cheat sheet through docs, CLI help, in-app help topics, and packaged release docs
- added `scripts/certify-release.sh` for clean-account release certification
- added `docs/RELEASE_CERTIFICATION.md` to define the release gate and QA note policy
- added snapshot-review summaries to CI and tag-release workflows
- release artifacts now point reviewers directly at `artifacts/tui-snapshots/index.html`

## Why this release exists

The app itself already had a strong writing flow.

The weak spots were around the edges:

- new users still had to infer too much from the menu bar
- release validation was real, but not recorded in a reusable way
- the nostalgia UI had snapshot coverage, but reviewers had to know where to look

`v2.2.0` closes those gaps without bloating the journal surface.

## Release QA Notes

- local pre-release validation hit low disk space once during debug compilation with only `117 MiB` free on `/System/Volumes/Data`
- the issue was resolved by running `cargo clean`, then rerunning validation successfully
- this was an environment constraint, not a product defect
