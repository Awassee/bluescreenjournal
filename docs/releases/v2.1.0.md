# v2.1.0

This is the clean `2.1` release line for BlueScreen Journal.

It consolidates the current product into one stable release with:

- the nostalgic blue-screen editor flow fully centered
- encrypted local-first journaling with append-only history
- real index, calendar, search, spellcheck, backups, verify, and sync
- direct cloud connector setup inside the product
- hardened installer and release validation paths

## Highlights

- `ENTRY NO.` now advances with each saved revision, and fresh pages show the next serial before you save
- the editor shell now makes page context easier to read:
  - `NEXT ENTRY`
  - `NEXT DAY`
  - `OLD ENTRY`
  - `SAVED PAGE`
- command palette labels are action-first and searchable by user intent
- first-run guidance now teaches:
  - `F2` to save this page
  - `**save**` to save and open the next same-day entry
  - `Alt+N` to open the next blank day
  - Calendar / Index / Search for older entries
- sync defaults and direct Google Drive / Dropbox connector setup are consistent across TUI, CLI, doctor, and trust surfaces
- direct Google Drive and Dropbox credentials can be stored in macOS Keychain
- installer launch and PATH integration are cleaner and less confusing

## Release engineering

- added a public one-line installer smoke script:
  - `./scripts/smoke-public-install.sh`
- release automation now smoke-tests the public installer against the pushed tag
- release packaging includes refreshed docs, screenshots, completions, man page, and Homebrew formula template
- the public bootstrap installer now selects the correct published macOS release asset instead of assuming an outdated universal filename
