# BlueScreen Journal v0.1.6

This release is a writing-surface refinement pass.

It keeps the vault, crypto, sync, and packaging model intact, but makes the product faster to read, navigate, and operate from inside the nostalgic blue-screen UI.

## Install

Turnkey macOS install:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash
```

Pin this release:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash -s -- --version v0.1.6
```

## Highlights

- added a DOS-style ruler above the editor with live cursor-column emphasis
- added live document stats in the footer so you can track writing progress without leaving the screen
- added `TOOLS -> Status Dashboard` for lock state, integrity state, sync target, entry stats, and vault summary
- added `FILE -> Backup Cleanup Preview` so retention behavior is visible before pruning
- improved the calendar with typed `YYYY-MM-DD` jump input and a clearer today highlight
- improved the index with live filter input and a newest-first / oldest-first sort toggle
- improved global search so results refresh while query and date filters change
- added selected-result details in the search overlay so opening a match is less blind

## Why this matters

- the editor now feels closer to a dedicated DOS-era word processor instead of a generic fullscreen terminal pane
- navigation costs less because date jump, index filtering, and search all work with fewer mode switches
- product trust improves because operational status and backup retention are visible inside the app, not just through CLI commands

## User-facing changes

- `F3` calendar now supports typed date jump in addition to grid navigation
- `F5` search feels more immediate because it reruns as inputs change
- `F7` index is easier to use on larger vaults because it supports typed filtering and sort toggling
- `Esc` menu flows now surface more operational insight without leaving the writing surface
