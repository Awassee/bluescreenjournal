# v2.1.1

This is a focused patch release on top of `v2.1.0`.

It does not try to widen the product. It tightens the installer and release experience so the first-run path feels more trustworthy and easier to follow.

## Highlights

- smart installer mode now explains what it chose:
  - bundled prebuilt
  - downloaded stable prebuilt
  - source install/update
- smart installer mode now prints a short execution plan before it starts work
- post-install option `1` now shows explicit handoff state before entering the full-screen app:
  - resetting terminal input state
  - handing off to `bsj`
  - opening the blue-screen workspace
- installer menu wording is more accurate:
  - `Print quickstart + key cheat sheet`
- release smoke coverage now exercises:
  - top-level smart installer option `1`
  - same-terminal launch handoff messaging

## Why this release exists

The main issue after `v2.1.0` was not core journaling behavior. It was confidence in the installer.

Option `1` in the installer felt too silent in a few paths, which made it look stuck even when it was progressing correctly. This patch makes the installer more explicit about:

- what mode it selected
- what it is about to do
- when it is handing off to the full-screen TUI
