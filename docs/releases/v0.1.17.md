# BlueScreen Journal v0.1.17

This release sharpens the product surface around update trust, installer reliability, and day-one discoverability.

If you want the short version:

- in-app update flow is now actionable from `TOOLS -> Check for Updates`
- About and soundtrack controls are easier to find from menus
- installer reruns are less likely to leave users on stale binaries
- installer now has built-in troubleshooting and PATH repair actions

## Highlights

### 1) In-app updater now feels real

`TOOLS -> Check for Updates` can now launch an update install action directly from inside the TUI and report status clearly.

Why this matters:
- less context switching out of the app
- clearer update intent and state
- fewer “did it actually update?” moments

### 2) About + audio controls are easier to discover

- added explicit `HELP -> About BlueScreen Journal`
- added clearer soundtrack controls in `TOOLS`:
  - `Soundtrack Source`
  - `Toggle Soundtrack`
- soundtrack auto-start now runs when a source is configured

Why this matters:
- first-run discoverability is better
- users can confirm version/credits/contact in one obvious place
- audio behavior is clearer and more consistent

### 3) Installer update behavior is more reliable

Installer reruns on machines that already have `bsj` now default to latest `main` source update flow (instead of silently appearing stale).

Installer now also prints:
- installed `bsj --version`
- active `PATH` binary location

Why this matters:
- easier to verify what build is actually running
- easier to debug stale PATH precedence issues

### 4) Installer now has built-in support actions

Added installer actions:
- `--doctor` (environment diagnostics + troubleshooting hints)
- `--repair-path` (shell profile PATH repair)
- `--about` (copyright/contact/about)

Why this matters:
- faster recovery from install/setup confusion
- fewer manual support loops for common shell/path problems

## Install / Update

Install or update with the turnkey command:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash
```

Pin this exact stable release:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash -s -- --prebuilt --version v0.1.17
```

Optional installer troubleshooting:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash -s -- --doctor
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash -s -- --repair-path
```

## Quick verification

```bash
which -a bsj
```
