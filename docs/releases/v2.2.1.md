# v2.2.1

This release is a confidence patch.

It does not try to add a new product identity. It tightens the trust, onboarding, and release-communication surfaces around the journal that already exists.

## Highlights

- turned `TOOLS -> Status Dashboard` into a real drill-down picker:
  - trust snapshot
  - verify and integrity details
  - backup snapshot and backup history
  - sync center and cloud status
  - cloud recovery
  - doctor report
- added `HELP -> What's New`
  - CLI mirror: `bsj guide whatsnew`
  - docs mirror: [docs/WHATS_NEW.md](../WHATS_NEW.md)
- kept the save/next-entry guidance explicit immediately after the first successful save
- shortened the installer post-install summary so it feels like a quick assist instead of a manual excerpt
- extended release certification to verify the new `whatsnew` guide surface

## Why this release exists

The writing surface was already strong.

The friction was around confidence and orientation:

- users could save, but the next move was not always obvious
- the dashboard had useful information, but not the right interaction shape
- release highlights were visible on GitHub, not inside the product

`v2.2.1` fixes those edges without bloating the editor.
