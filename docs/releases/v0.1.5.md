# BlueScreen Journal v0.1.5

This release is an operator-trust and contributor-readiness pass.

It does not change the core journal model. It makes the product easier to operate, debug, evaluate, and contribute to without leaving the repo or guessing at the intended workflows.

## Install

Turnkey macOS install:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash
```

Pin this release:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash -s -- --version v0.1.5
```

## Highlights

- added dedicated operator guides for:
  - troubleshooting
  - sync
  - backup and restore
  - macros
  - terminal behavior
  - privacy and data handling
- added project-management docs for roadmap and contributing
- added a GitHub pull request template for cleaner contributor intake
- expanded built-in guide coverage so the binary can print the new operator docs directly:
  - `bsj guide troubleshooting`
  - `bsj guide sync`
  - `bsj guide backup`
  - `bsj guide macros`
  - `bsj guide terminal`
  - `bsj guide privacy`
- improved release-bundle smoke tests so the new docs are verified in packaged installs

## Why this matters

- operators now have a clearer path when sync, verify, restore, or terminal behavior goes wrong
- privacy boundaries are documented in plain language instead of being implied by the codebase
- macro behavior is documented as a safe workflow tool rather than an open-ended scripting surface
- contributors now have a roadmap, a contribution guide, and a PR template instead of learning the repo purely by inference
