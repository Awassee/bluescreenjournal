# BlueScreen Journal v0.1.14

This is a reliability patch release for the public installer path.

It targets a real-world install failure seen on a fresh machine and hardens the bootstrap handoff behavior under strict shell settings.

## Install

Turnkey macOS install:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash
```

Pin this release:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash -s -- --version v0.1.14
```

## Fixes

- fixed `install.sh` failures under `set -u` when no install override args are present
- hardened bootstrap argument forwarding for both:
  - bundled prebuilt installer delegation
  - bundled source installer delegation
- kept behavior unchanged when overrides are provided, while making the empty-override path explicit and safe

## Validation improvements

- added a smoke-test regression path that runs bootstrap install with no forwarded install args
- retained existing bundle install checks, bootstrap checks, docs/man/completion checks, and binary audit checks

## Why this matters

- the one-line public install path is now resilient to empty-argument edge cases on strict Bash environments
- installer behavior is predictable whether or not install overrides are passed
