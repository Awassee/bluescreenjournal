# v1.3.2

This patch release focuses on shipping a more coherent first-run product: clearer save flow, clearer next-entry vs next-day behavior, cleaner installer launch, and consistent direct-cloud setup.

## Highlights

- `ENTRY NO.` now advances with each saved revision, and a fresh page shows the next serial before you save
- the editor now distinguishes:
  - `NEXT ENTRY` for a clean same-day follow-up page
  - `NEXT DAY` / `BLANK DAY` when you deliberately move forward in time
  - `OLD ENTRY` when you open archive history
- command palette labels are now action-first and searchable by intent words like:
  - `save`
  - `next`
  - `old`
  - `search`
  - `backup`
  - `help`
- first-run guidance now clearly teaches:
  - `F2` to save this page
  - `**save**` to save and open the next same-day entry
  - `Alt+N` to open the next blank day
  - Calendar / Index / Search for older entries
- save receipts and header/footer status now make saved state and next actions easier to read
- sync backend selection is now consistent across CLI, TUI, doctor, and settings surfaces
- `SETUP -> Cloud Provider Setup` now covers both:
  - folder-based provider sync
  - direct Google Drive / Dropbox API modes
- Dashboard, Journal Health, Sync Center, and Cloud Status now show:
  - effective sync mode
  - whether that mode came from env override or saved Setup defaults
  - clearer connector readiness status
- installer launch now prefers a clean same-terminal handoff, with an explicit separate option to open a fresh Terminal window
- installer completion output is more explicit about menu-first setup and cloud configuration
- direct Google Drive and Dropbox connector credentials can be saved in macOS Keychain from inside the TUI

## UX polish

- top menus are grouped more cleanly:
  - spellcheck lives under `TOOLS`
  - updates and doctor live under `HELP`
- footer wording now uses clearer labels like `STATE`, `WORDS`, `LINE`, and `COL`
- the always-visible shell now keeps the DOS-era feel while making page state more obvious at a glance
- old-entry browsing remains intentionally deliberate to preserve the journal’s “today-first” feel

## Config additions

The app now supports saved sync defaults in config:

- `sync_backend_preference`
- `gdrive_folder_id`
- `dropbox_root`

Secrets still stay out of config by default. Direct connector credentials can now live in macOS Keychain instead of config files.

## Operator impact

- `bsj doctor` no longer treats `gdrive` and `dropbox` as unknown backends
- `bsj sync`, `bsj cloud status`, `bsj cloud recover`, and `SYSOP -> Sync Preview` can use saved Setup defaults when `--backend` and `BSJ_SYNC_BACKEND` are not provided
- direct connector auth now reports whether it is using env vars, Keychain, or both
- smoke install coverage now guards the new post-install cloud setup guidance
- release smoke now covers the post-install menu numbering and launch wording
