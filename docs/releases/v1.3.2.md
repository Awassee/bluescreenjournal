# v1.3.2

This patch release focuses on sync clarity and setup consistency.

## Highlights

- sync backend selection is now consistent across CLI, TUI, doctor, and settings surfaces
- `SETUP -> Cloud Provider Setup` now covers both:
  - folder-based provider sync
  - direct Google Drive / Dropbox API modes
- Dashboard, Journal Health, Sync Center, and Cloud Status now show:
  - effective sync mode
  - whether that mode came from env override or saved Setup defaults
  - clearer connector readiness status
- installer launch is more resilient if PTY launch fails
- installer completion output is more explicit about menu-first setup and cloud configuration
- direct Google Drive and Dropbox connector credentials can be saved in macOS Keychain from inside the TUI

## Config additions

The app now supports saved sync defaults in config:

- `sync_backend_preference`
- `gdrive_folder_id`
- `dropbox_root`

Secrets still stay out of config by default. Direct connector credentials can now live in macOS Keychain instead of config files.

## Operator impact

- `bsj doctor` no longer treats `gdrive` and `dropbox` as unknown backends
- `bsj sync`, `bsj cloud status`, `bsj cloud recover`, and `SYSOP -> Sync Preview` can use saved Setup defaults when `--backend` and `BSJ_SYNC_BACKEND` are not provided
- direct connector auth now reports whether it is using env vars, Keychain, or both
- smoke install coverage now guards the new post-install cloud setup guidance
