# BlueScreen Journal v0.1.10

This release closes the remaining “read-only admin screen” gaps in the TUI.

The app already had the underlying export, restore, and sync machinery. The weak spot was that a
few history and status surfaces stopped at information when they should have carried the user
directly into the next action.

## Install

Turnkey macOS install:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash
```

Pin this release:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash -s -- --version v0.1.10
```

## Highlights

- export history now reopens a live export prompt with the saved path and format prefilled
- backup history now opens restore with the chosen encrypted snapshot already selected
- Sync Center is now a real control surface with direct actions for sync, sync history, diagnostics,
  settings, integrity review, and a detailed sync snapshot
- the new flows are covered by targeted TUI tests instead of depending only on manual exercise

## Why this matters

- history screens now finish the job instead of forcing the user back through another menu
- restore and export workflows are faster because they preserve prior operator choices
- sync administration feels more like a coherent product surface and less like a detached status
  panel

## User-facing changes

- `FILE -> Export History` becomes a reusable export preset picker
- `FILE -> Backup History` becomes a restore launcher for encrypted snapshots
- `TOOLS -> Sync Center` becomes an action menu with preview, history, doctor, settings, and
  integrity options
