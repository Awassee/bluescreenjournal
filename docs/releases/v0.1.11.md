# BlueScreen Journal v0.1.11

This is a backup-safety patch release.

The issue was narrow but important: encrypted backups were named with only second-level timestamp
precision, so two snapshots created within the same second could collide on disk. That could
overwrite the earlier snapshot before retention policy even had a chance to run.

## Install

Turnkey macOS install:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash
```

Pin this release:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash -s -- --version v0.1.11
```

## Fixes

- backup filenames now use subsecond precision when needed, which prevents same-second collisions
- backup creation now checks for an existing path and advances the timestamp token before writing
- backup timestamp parsing remains compatible with older second-only backup filenames
- added regression tests for rapid backup creation and mixed legacy/fractional backup timestamps

## Why this matters

- manual snapshots are now append-safe even when created quickly back-to-back
- restores continue to work for older backup files already sitting in existing vaults
