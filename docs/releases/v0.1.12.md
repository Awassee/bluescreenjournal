# BlueScreen Journal v0.1.12

This is a performance release.

The changes are focused on the TUI hot paths that run constantly while you type, search, and move
through picker overlays. The goal was simple: reduce unnecessary work in redraw-heavy paths without
changing the product surface or storage model.

## Install

Turnkey macOS install:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash
```

Pin this release:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash -s -- --version v0.1.12
```

## Improvements

- picker overlays now cache lowercase filter haystacks and filtered index lists instead of rebuilding
  them on every redraw and cursor movement
- live document stats now use a cached model, so footer counts and goal progress stop reconstructing
  the full entry text repeatedly
- the text buffer now computes lines, words, and characters directly from in-memory lines without
  extra string joins or temporary allocations
- TUI tests were updated so direct buffer replacement paths refresh stats correctly after edits,
  loads, and recovery flows

## Why this matters

- command palette, search history, and other picker surfaces feel more responsive while typing
- regular screen redraws do less work in long entries
- the editor keeps the same behavior, but with lower overhead in the paths users hit most
