# BlueScreen Journal v0.1.9

This release is a production-readiness pass on the TUI itself.

It focuses on the places where a terminal product either feels finished or obviously unfinished:
screen fit, first-run clarity, recovery clarity, and whether the important prompts still work when
the UI gets dense.

## Install

Turnkey macOS install:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash
```

Pin this release:

```bash
curl -fsSL https://raw.githubusercontent.com/Awassee/bluescreenjournal/main/install.sh | bash -s -- --version v0.1.9
```

## Highlights

- fixed popup sizing so major overlays no longer clip their bottom instructions
- improved the setup wizard with step-specific guidance for vault path, passphrase, and entry-number epoch
- improved the unlock prompt with vault-path context and Keychain-memory guidance
- made export prompts clearer that exports are plaintext and unsaved edits are included
- made restore prompts clearer that restores should target a new or empty folder
- improved conflict prompts with better date/head context and clearer selection guidance
- kept a compact `Ctrl+K` commands hint when the full footer legend is hidden
- added render-smoke tests for help, settings, search, index, restore, sync, conflict, and small-terminal warning screens

## Why this matters

- the app is safer to trust because recovery and export surfaces are more explicit
- the UI is easier to use because critical hints are no longer clipped off-screen
- the DOS-style interface feels tighter because dense screens now fit the 80x25 contract more cleanly
- regressions are harder to reintroduce because major screens now have render coverage, not just logic coverage

## User-facing changes

- `F1` help is denser and now fits the classic workspace cleanly
- setup and unlock flows explain what is happening instead of acting like raw input prompts
- conflict review shows the conflicted date, head count, and clearer selection controls
- index, search, sync, restore, and settings prompts now keep their bottom action hints visible
- quick exports now announce themselves as plaintext exports in status feedback
