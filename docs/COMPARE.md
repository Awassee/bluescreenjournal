# Compare bsj

## bsj vs plaintext journal folders

| Question | bsj | Plaintext folder |
| --- | --- | --- |
| Encrypted at rest by default | Yes | No |
| Append-only revisions | Yes | Usually no |
| Encrypted autosave drafts | Yes | Usually no |
| Built-in date index and calendar | Yes | No |
| No plaintext search index on disk | Yes | Usually no index or a plaintext one |
| Menu-driven terminal UI | Yes | No |

## bsj vs modern GUI note apps

| Question | bsj | GUI note app |
| --- | --- | --- |
| Mouse-free terminal workflow | Yes | Usually no |
| Local-first encrypted vault control | Yes | Varies |
| Deliberate DOS-style focused writing surface | Yes | Rarely |
| Cloud account required | No | Often |
| Broad collaboration/project features | No | Often yes |

## bsj vs PKM systems

bsj is not trying to win on graph features, plugins, or knowledge management sprawl.

It is stronger when you want:

- a dedicated journaling workflow
- date-based writing and retrieval
- strict terminal feel
- encrypted local vaults and encrypted sync blobs

It is weaker when you want:

- backlinks and graph views
- publishing workflows
- rich-text layout
- collaborative editing
- mobile-first usage

## Who should choose bsj

Choose bsj if you want a private daily writing tool that feels like an old word processor but behaves like a modern encrypted local app.
